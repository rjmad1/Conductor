# ADR-008: Analytics Architecture

## Status
Approved

## Context
Conductor tenants require near real-time dashboards to monitor campaign status, deliverability metrics (sent, delivered, read, failed ratios), message costs, and workflow automation drop-off rates. 

Building custom chart widgets, SQL aggregators, export-to-CSV libraries, and complex reporting filters in the React dashboard frontend is an extensive task that risks code bloating. We need a design that provides advanced visualization capabilities while keeping engineering effort low.

## Decision
We will **Adopt Metabase** as our self-hosted business intelligence and analytics engine, utilizing its **Signed JWT Interactive Embedding** feature:
1.  **Read Replica Analytics:** Metabase is connected directly to a read-replica database instance of PostgreSQL. Live transactions on the primary database are not affected by heavy reporting queries.
2.  **Dashboard Configuration:** Campaign analytics dashboards and charts are designed visually inside Metabase by product managers or analysts.
3.  **Secure JWT Iframe Embedding:** The React dashboard frontend displays reporting charts by embedding Metabase in an iframe.
    *   The Spring Boot application server exposes a secure token endpoint.
    *   This endpoint generates a signed JSON Web Token (JWT) using a shared secret key shared with Metabase.
    *   The JWT payload embeds a locked parameter constraint: `tenant_id = :currentTenantId`.
    *   Metabase validates the JWT signature and dynamically filters the SQL queries to return only the current tenant's data.

## Alternatives Considered
1.  **Custom Frontend Charts (Recharts/Chart.js + custom APIs):** Rejected. Requires building custom SQL aggregators, pagination endpoints, CSV export controls, and layout builders from scratch, which slows MVP speed.
2.  **SaaS Analytics (Tableau / PowerBI):** Rejected due to cost. Subscription licenses for embedded viewers are highly expensive and raise vendor lock-in risks.

## Consequences
*   **Rapid Dashboard Iteration:** Analytics changes, new charts, or report layouts are adjusted in Metabase without requiring frontend redeployments or backend code changes.
*   **High Performance:** Read replicas separate analytical queries from operational transactional workloads.
*   **Secure Multi-Tenancy:** Metabase's locked parameters prevent a tenant from altering the iframe URL to view another tenant's metrics.

## Risks & Mitigations
*   **Risk:** Leakage of the shared JWT embedding secret or misconfiguring parameter locks in Metabase.
*   **Mitigation:** Rotate the JWT secret regularly using environment injection. Enforce compliance reviews ensuring that all parameters in embedded Metabase questions are marked as "Locked" (denying parameter modification from the frontend URL).
